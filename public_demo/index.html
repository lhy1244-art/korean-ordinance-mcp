경기도 노인 디지털 격차 해소

정책 주제 한 줄을 입력한 뒤, 4단계 도구가 만들어낸 실제 산출물을 모두 다운로드할 수 있습니다. 각 단계는 독립 호출 가능.

정책 의도 경기도 노인 디지털 격차 해소
성안 1 · 발굴

입법례 스크리닝

정책 의도 → 5개 관할(한·일·영·EU·미)에서 직접/간접 참고 입법례 자동 수집·요약

결과 (실제):
  • • 총 12건 수집 (4개 관할 통과 — JP는 관련성 필터로 제외)
  • • 직접 참고 3건: 대구 남구·북구·대전 서구 노인 생활디지털 교육 조례
  • • 간접 참고 9건: UK·EU·US
  • • 의사결정 가이드: 경기도 기존 조례 미검색 → 신규 제정 검토

성안 2 · 결정

결정 카드 추출

1단계 결과를 토대로 LLM이 *결정해야 할 항목*과 *선택지*를 뽑아 객관식으로 제시. 정치적 판단을 AI에 맡기지 않는 안전장치.

7개 결정 카드 추출:
  • • 지원 대상 연령 (3안)
  • • 지원계획 수립 주기 (3안)
  • • 교육 제공 장소·방식 (3안)
  • • 사무 위탁 가능 여부 (2안)
  • • ... + 영향분석지표 자동 평가

환각 방지: 각 선택지의 *근거 타지자체*는 1단계 카드에 실제 등장한 jurisdiction만 인용. 근거 없으면 정직하게 "근거 없음" 표시.

성안 3 · 만들기

조례 초안 작성

결정값과 1단계 모범사례를 LLM 컨텍스트로 넘겨 제정안 또는 일부개정안(신·구조문대비표 포함)을 자동 생성.

제정안 경기도 노인 디지털 격차 해소 지원 조례
  • • 조문 11개 (목적/정의/도지사 책무/지원대상/지원계획/지원사업/교육 운영/시·군 협력/위원회/재원/시행규칙 위임)
  • • 결정값 5개 모두 조문에 정확히 반영 ✅
  • • 대구 모범사례 차용 (3년 단위 지원계획, 인터넷뱅킹·금융사기 방지 교육, 경로당·복지관 다층 인프라)

성안 4 · 검토 (셀프)

자동 셀프 점검

초안 작성 직후 *별도 LLM 검증 에이전트*가 결정값 모순·환각 인용·표준 구조 누락 등을 환기. 본격 검토 트랙은 별개.

검증 에이전트가 발견한 모순 (실제):
"결정값은 대체로 반영되었으나, 제6조제2항 민간 위탁 허용이 '도 직영' 결정과 충돌 가능성(발의자 의도 재확인 필요)..."

작성자(정책지원관 페르소나)는 셀프 검토가 약하다는 실무 관찰에서 출발 — 도구가 자동으로 빠뜨림을 환기.

셀프 점검 섹션 (조례안 종류에 따라 다름):
  1. 결정값 반영 점검
  2. 상위법 명백한 저촉 (sanity)
  3. 인용 법률·조례 환각 의심
  4. 표준 구조 누락 (부칙·시행규칙 위임 등)
  5. 사전 입법영향분석지표 자가평가 (10항목 a~j 표)
  6. 오탈자·맞춤법 점검
  7. 개정안 정합성 점검 — 지시문 ↔ 신·구조문대비표 ↔ 본문 불일치 (개정안 한정)

실제 발견 사례 (개정안): 「제8조의2 신설로 기존 제8조가 제9조로 밀리는데, 신설 조문 내부 인용 ‘제8조에 따른’이 옛 번호 그대로 — *조문이 자기 자신을 가리키는 순환참조* 위험을 자동 적발」.

제정안용 5+1개 섹션 (1~6번)
